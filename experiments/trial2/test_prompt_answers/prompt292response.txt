User: can u explain what an event loop is in javascript, i keep hearing "non blocking" and nodding along

Assistant: JavaScript runs on a single thread, but the event loop lets it handle async tasks (like network requests) without freezing. When you call something async, JS hands it off (to the browser/Node runtime), keeps running other code, and once that async task finishes, its callback gets placed in a queue. The event loop constantly checks: is the main thread free? If yes, take the next item from the queue and run it. That's what "non-blocking" means - the main thread never sits idle waiting.
