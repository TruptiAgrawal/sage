**What is an Algorithm?**

An algorithm is a step-by-step procedure for solving a problem or performing a computation. Think of it like a recipe: if you follow the steps precisely, you'll get the desired result. Algorithms are fundamental to computer science—they're the thinking behind programs.

**Why Algorithms Matter:**

1. **Correctness:** Algorithms ensure solutions are correct
2. **Efficiency:** Different algorithms solve the same problem with different efficiency
3. **Scalability:** Good algorithms work efficiently even with large amounts of data
4. **Foundation:** Understanding algorithms helps you write better code
5. **Problem-solving:** Algorithm design is problem-solving thinking

**Core Characteristics of Algorithms:**

- **Input:** Data the algorithm receives
- **Output:** Result the algorithm produces
- **Finiteness:** Must terminate (not loop forever)
- **Definiteness:** Each step precisely defined
- **Effectiveness:** Each step must be doable

**Common Types of Algorithms:**

1. **Sorting Algorithms** (arranging data in order)
   - Bubble sort: Simple but slow
   - Merge sort: Efficient, "divide and conquer"
   - Quick sort: Fast on average
   - Heap sort: Consistent performance

2. **Searching Algorithms** (finding items)
   - Linear search: Check each item one by one
   - Binary search: Divide and eliminate half each time (faster)

3. **Graph Algorithms** (working with networks/connections)
   - Breadth-first search: Explore layer by layer
   - Depth-first search: Explore deep paths first
   - Dijkstra's: Find shortest path
   - A*: Pathfinding with heuristics

4. **Dynamic Programming** (breaking problems into subproblems)
   - Fibonacci sequence calculation
   - Longest common subsequence
   - Knapsack problem

5. **Greedy Algorithms** (make locally optimal choices)
   - Huffman coding: Efficient data compression
   - Activity selection: Maximum non-overlapping activities
   - Dijkstra's: Shortest path

6. **Divide and Conquer** (break into smaller problems)
   - Merge sort
   - Quick sort
   - Binary search

**Algorithm Complexity Analysis:**

One of the most important aspects is understanding how algorithms scale:

**Time Complexity:** How fast does the algorithm run?
- O(1): Constant time - very fast regardless of input size
- O(log n): Logarithmic - doubles input, adds one step
- O(n): Linear - time proportional to input size
- O(n log n): Linearithmic - efficient for sorting
- O(n²): Quadratic - slow for large inputs
- O(2ⁿ): Exponential - very slow, impractical for large inputs

**Space Complexity:** How much memory does it use?
- Similar notation: O(1), O(n), O(n²), etc.

Example: Searching in a list of 1 million items:
- Linear search: 1,000,000 operations worst case
- Binary search: ~20 operations (log₂ 1,000,000 ≈ 20)

**Real-World Example: Sorting**

Let's look at sorting because it illustrates key concepts:

Bubble Sort (Simple but inefficient):
```
compare each pair
if first > second, swap them
repeat until no swaps needed
```
Time complexity: O(n²) - slow for large lists

Merge Sort (Efficient):
```
divide list in half
sort left half
sort right half
merge the sorted halves
```
Time complexity: O(n log n) - much faster

For 1 million items:
- Bubble sort: ~1 trillion operations
- Merge sort: ~20 million operations
That's 50,000x faster!

**Algorithm Design Techniques:**

1. **Brute Force:** Try all possibilities
   - Simple but often too slow

2. **Divide and Conquer:** Break into smaller problems
   - Merge sort, quicksort, binary search

3. **Greedy:** Make locally optimal choices
   - Huffman coding, activity selection

4. **Dynamic Programming:** Remember previous solutions
   - Fibonacci, longest path

5. **Backtracking:** Try possibilities, backtrack if wrong
   - Solving mazes, Sudoku, N-queens problem
