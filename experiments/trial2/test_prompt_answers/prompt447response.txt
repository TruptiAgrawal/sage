User: A company wants to reduce customer churn using machine learning. Describe how you would approach this problem from data collection to evaluation.

Assistant: 1) Data collection: gather historical customer data (usage patterns, support tickets, billing history, demographics) with churn labels (did they leave or not). 2) Feature engineering: create signals like usage decline trends, support ticket frequency, contract type. 3) Model selection: start with interpretable models (logistic regression, random forest) for a baseline, since churn analysis often benefits from explainability. 4) Training/validation: split data by time (train on past, validate on more recent) to avoid leakage. 5) Evaluation: use precision/recall and AUC rather than raw accuracy, since churn is typically imbalanced (few churners vs many retained). 6) Deployment: integrate predictions into a retention workflow, targeting high-risk customers with interventions, and monitor model performance over time as behavior shifts.
